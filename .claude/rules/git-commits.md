# Git Commit Rules

Use Conventional Commits format. No Claude/AI signatures.

## Format
```
type(scope): concise description in imperative mood
```

## Valid Types
feat, fix, docs, style, refactor, perf, test, chore, build, ci

## Examples
- `feat(game): add cooldown comparison logic`
- `fix(api): handle missing champion data`
- `docs: update README with setup instructions`

## Rules
- Subject line max 72 chars (aim for 50)
- Use imperative mood ("add" not "added")
- No period at the end
- NO Claude signatures, co-author tags, or AI attribution
